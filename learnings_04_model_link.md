# 学び 04: モデル管理の仮想化（シンボリックリンクの活用）

## 状況
- `ilab model download` を実行した際、モデルの実体が OS 標準のキャッシュディレクトリ (`~/.cache/instructlab/models/`) に保存された。
- リポジトリ内 (`~/instructlab-granite4/models/`) を空のままにせず、プロジェクトのポータビリティを維持したい。

## 対策: シンボリックリンクによる「窓」の作成
- ファイルを物理的にコピーするのではなく、Linux の `ln -s` コマンドを使用してシンボリックリンクを作成。
- **実行コマンド:**
  `ln -s /home/matsuo/.cache/instructlab/models/granite-4.0-1b-Q4_K_M.gguf ~/instructlab-granite4/models/`

## メリット
1. **ディスク容量の節約:** 1GB超のモデルファイルを二重持ちせず、16GB RAM環境の貴重なリソースを保護。
2. **パスの簡略化:** `config.yaml` 内の記述を `/home/matsuo/instructlab-granite4/models/...` とプロジェクト相対のように記述でき、可読性が向上。
3. **管理の統一:** 設定ファイル (config) とモデルへの参照 (link) が同じリポジトリ内に存在するため、構成が把握しやすい。

## 結論
ツール固有のデフォルト挙動（キャッシュ保存）を受け入れつつ、シンボリックリンクを「窓」として使うことで、システム構成のクリーンさとプロジェクトの独立性を両立させた。
